# 晚霞预报 · 小红书发帖 SOP

> 自动发帖停用（小红书风控检测）。素材生成 → 发帖全部手动。
> 最后更新：2026-06-29

## 两步走

### 1. 生成素材包（电脑，5 分钟）

```bash
cd E:\x-tool\wanxia
start-wanxia.bat                    # 终端1：启动服务（保持）
node scripts/publish-xhs.js         # 终端2：生成素材包（跑完可关）
```

跑完后 `posts/` 下出现当日目录：

```
posts/2026-06-29/
├── posts.json              # 索引（标题/封面/POI 都在里面）
├── 01-national.png / .txt  # 全国播报
├── 02-杭州.png / .txt      # 城市帖
├── 03-广州.png / .txt
├── ...
└── covers/                 # AI 封面图（首图用这个）
```

> 💡 如果电脑中午常开，可以把 `start-wanxia.bat` 一直开着——cron 每天 12:00 自动跑 `publish-xhs.js`，跳过这步，直接进入第 2 步。

### 2. 手机发帖（15 分钟）

一条一条来，间隔 3-5 分钟：

| 步骤 | 操作 |
|------|------|
| 首图 | 选 `covers/` 下的 AI 封面图 |
| 第二张 | 选同编号的 `.png` 截图 |
| 标题 | 从 `posts.json` 复制对应帖子的 `title` |
| 正文 | 打开同名 `.txt`，全文复制 |
| 话题 | 文案末尾已自带，不用手动加 |
| POI | 全国帖不加；城市帖加地标 |
| 发布 | 检查标题≤20字、封面图正常 → 发布 |

**发布后快速检查**：封面加载正常 / 正文格式完整 / POI 正确 / 5 个标签以内。

## 异常

| 情况 | 做法 |
|------|------|
| `publish-xhs.js` 超时 | 确认 `start-wanxia.bat` 的终端没关（脚本连 localhost:8080） |
| 封面图缺失（`cover: null`） | 用正文截图代替首图 |
| 敏感词审核不通过 | 替换「翻车」→「没戏」、「极佳」→「高概率」，重发 |
| 当天城市全低分 | 至少发全国播报帖一条 |

## 速查

```
电脑: start-wanxia.bat → node scripts/publish-xhs.js (5min)
手机: 打开小红书 App → 照着素材包逐条发 (15min)
素材: wanxia/posts/YYYY-MM-DD/
```
