## 〇、第一步：获取当天的素材包

**`publish-xhs.js` 需要连接 wanxia 服务（localhost:8080）才能截图**——所以服务必须先跑起来。

### 模式 A：让 cron 自动生成（推荐——设一次就忘）

每天中午 12:00，server.js 的 cron 自动执行 `publish-xhs.js`，生成当日素材包。

```bash
cd E:\x-tool\wanxia
start-wanxia.bat          # 或 node server.js
```

**保持这个终端窗口开着就行。** 每天 12:00 自动在 `posts/YYYY-MM-DD/` 下产出素材包，你只需要在方便的时候来手动发帖。

> 验证 cron 是否在跑：中午 12:05 检查 `ls posts/$(date +%Y-%m-%d)/`，有文件就说明 cron 正常。

### 模式 B：手动触发（适合偶尔发一次）

1. 先启动服务：
```bash
cd E:\x-tool\wanxia
start-wanxia.bat          # 终端 1：启动服务（保持开着）
```

2. 另开终端，手动生成素材包：
```bash
cd E:\x-tool\wanxia
node scripts/publish-xhs.js   # 终端 2：生成素材包（约 3-5 min）
```

> ⚠️ `publish-xhs.js` 会启动 Playwright 浏览器连接 `localhost:8080` 截图。如果服务没启动，脚本会卡在「加载前端」步骤超时。

## 一、素材包结构

每天管线产出目录 `posts/YYYY-MM-DD/`，包含：

```
posts/2026-06-22/
├── posts.json           # 索引文件 — 所有帖子的元数据
├── 01-national.png      # 全国等高线概览截图
├── 01-national.txt      # 全国播报文案
├── 02-杭州.png          # 城市详情截图
├── 02-杭州.txt          # 城市专属文案
├── 03-广州.png / .txt
├── 04-厦门.png / .txt
├── 05-北京.png / .txt
├── 06-上海.png / .txt
└── covers/              # AI 生成的封面图（GPT-Image-2 摄影级）
    ├── 01-national.png
    ├── 02-杭州.png
    └── ...
```

### posts.json 字段说明

```json
[
  {
    "index": 1,
    "city": "national",
    "title": "📸 今日晚霞地图 · 06-22",
    "image": "01-national.png",
    "cover": "covers/01-national.png",
    "poi": null,
    "score": 74,
    "description": "..."
  }
]
```

| 字段 | 用途 |
|------|------|
| `title` | 小红书标题（≤20字，含 emoji，已适配平台限制） |
| `image` | 正文截图（等高线图/城市详情图） |
| `cover` | AI 封面图（首图，吸引点击） |
| `poi` | 地理位置标记（`null` 表示不加 POI，城市帖带 lat/lon） |

## 二、发帖前准备

### 2.1 确认管线已运行

```bash
# Windows PowerShell / WSL
ls E:\x-tool\wanxia\posts\$(date +%Y-%m-%d)\
```

如果当天目录不存在或为空，手动触发管线：

```bash
cd E:\x-tool\wanxia
node scripts/publish-xhs.js
```

耗时约 3-5 分钟（生成截图 + AI 封面图）。

### 2.2 登录小红书

**推荐用手机 App**（桌面端网页版功能受限）：
1. 打开小红书 App
2. 确认登录账号为「晚霞预报」账号
3. 保持在首页，准备发帖

> ⚠️ 如果用桌面浏览器：登录后保持标签页打开，不要刷新。

## 三、逐帖发布步骤

### 第一条：全国播报帖（置顶/当日主帖）

1. **打开素材包目录**：找到 `posts/YYYY-MM-DD/`
2. **选择首图**：用 `covers/01-national.png`（AI 封面，上部留白+摄影感）
3. **点击小红书 "+" 发帖**
4. **选择图片**：
   - 第一张：`covers/01-national.png`（封面图）
   - 第二张：`01-national.png`（等高线截图）
5. **填写标题**：从 `posts.json` 中取 `title` 字段，直接复制
6. **填写正文**：打开 `01-national.txt`，全选复制粘贴
7. **添加话题标签**：文案末尾已有（`#晚霞预报 #夏日晚霞 ...`），无需手动加
8. **POI 定位**：全国帖不加 POI（`poi: null`）
9. **预览检查**：
   - [ ] 标题 ≤20 字
   - [ ] 封面图是 AI 生成的摄影感首图
   - [ ] 正文有互动引导（「评论区晒图👇」）
   - [ ] 话题标签 5 个以内
10. **发布**

### 第二条起：城市详情帖

以杭州为例（`02-杭州`）：

1. **首图**：`covers/02-杭州.png`
2. **第二张**：`02-杭州.png`（杭州区域等高线 + 分数）
3. **标题**：从 `posts.json` 中取杭州帖的 `title`
4. **正文**：打开 `02-杭州.txt`，复制全文
5. **POI**：加杭州地标（如「西湖」「钱塘江」），搜索后选择
6. **发布**

依次处理 `03-广州`、`04-厦门`、`05-北京`、`06-上海`……直到当天城市帖发完。

> 💡 每天通常 5-6 条（1 条全国 + 4-5 条城市）。发布间隔建议 **3-5 分钟**，太密可能触发频率限制。

## 四、发帖后的验证清单

| 检查项 | 操作 |
|--------|------|
| 封面图加载正常 | 退出 App 重新进入，查看自己主页 |
| 标题完整显示 | 确认没有被截断 |
| 正文格式正确 | 换行、emoji、标签均正常 |
| 第二张图（截图）可见 | 左右滑动查看 |
| POI 正确 | 城市帖应该有定位标记 |
| 评论区互动 | 有评论时及时回复（提升权重） |

## 五、异常处理

### 5.1 发布失败（网络超时 / 审核不通过）

1. 等待 5 分钟后重试
2. 如果审核不通过：删除原文 → 检查文案是否有敏感词（「翻车」「极佳」→ 替换为「没戏」「高概率」）→ 重新发布
3. 连续 3 次失败：跳过该条，记录到 `posts/failed/`，次日手动补发

### 5.2 封面图生成失败

`posts.json` 中 `cover: null` 表示 AI 封面未生成：
- 用正文截图 `01-national.png` 作为首图
- 次日联系管理员检查 GPT-Image API Key

### 5.3 当天没有值得发的城市（全部 < 50 分）

`posts.json` 可能只有 1-2 条：
- 至少发全国播报帖（标题改为「今日晚霞预测 · 待观察」）
- 如果分低到只剩 0 条，参见 `posts.json` —— 管线会自动判断最低发布阈值
